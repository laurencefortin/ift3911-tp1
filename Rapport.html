Hypothèses

  • Général
    • Pour les consultations, les clients peuvent seulement chercher par aéroport/port/gare
    • Pour les consultations des administrateurs, il est possible de chercher par aéroport/port/gare mais également par compagnie.
    • Quand on arrive au menu, on choisi le type de voyage qu'on cherche pour ensuite choisir l'aéroport/port/gare pour finalement selectioner les autres critères
    • En se qui concerne le menu, il retourne une liste du choix prédéfini.
    • Même si les voyages férroviaires et aérien sont hautement similaire en terme de verification, on sépare quand même le menu en 2 sections comme le voyageur connait ses préférences.
    • Réservation de siège avec préférences est une classe
    • Pour des raisons éthiques, le systeme ne mémorise jamais les numéros de carte de crédit pour usage future.
    • Quand l'administeur recherche par compagnie, il doit choisir un choix du menu : Création, Modification, Supression
  • Voyage aérien:
    • Les sections des avions vont toujours être celles-ci :Première (F), Affaire (A), Économique Premium (P), Économique (E)
    • Une section est une classe
    • On affiche le nombre de siège disponible par section, dans un monde idéal, on affiche le layout de l'avion pour que l'utilisateur choisi le siege désiré
    • Il y a pas de limite dans les avions, quand on crer un vol l'aeroport associé s'assure de fournir un avion pour ce dit vol.
  • Voyage naval:
    • On assume que 1 cabine = 1 personne donc 1 réservation
    • Une section est une classe
    • Le port d'origine est le même que le port d'arrivé, c'est-à-dire qu'il n'y a pas de destination dans la recherche
  • Voyage ferroviaire:
    • Une ligne de train est une compagnie
    • Un trajet est un itinéraire

Contraintes OCL

Context Compagnie
inv: Compagnie.allInstances()->forAll(a1,a2|a1<>a2 implies a1.id <> a2.id)
inv: self.id.Length() < 6

Context Voyage
inv: self.DateDepart < self.DateArrivee
inv: self.HeureDepart < self.HeureArrivee

Context Vol
inv: Vol.allInstances()->forAll(b1,b2|b1<>b2 implies b1.id <> b2.id)
inv: self.EndroitDepart <> self.EndroitArrivee

Context Avion
inv: self.rangee <= 100
inv: self.colonne <= 10

Context Port
inv: Port.allInstances()->forAll(c1,c2|c1<>c2 implies c1.id <> c2.id)

Context Siege
inv: Siege.allInstances()->forAll(c1,c2|c1.section = c2.section implies c1.taux = c2.taux)
inv: Siege.paid and Siege.contains(Reservation) implies siege.contains(Confirmation)
Context Siege::reserver(a : client)
pre: self.contains(Reservation) -> excludes(a)
post : reserver(a) implies self.contains(Reservation)
Context siege::confirmer(a : client)
pre: not(self.contains(Reservation ) and self.Reservation.Contains(a)) -> excludes(a)
post : self.contains(Confirmation) and self.Confirmation.Contains(a)

Context Gare
inv: Gare.allInstances()->forAll(d1,d2|d1<>d2 implies d1.id <> d2.id)

Context Croisiere
inv: (self.DateArrivee - self.DateDepart) < 21
inv: self.ArraylistPort.Last() = self.EndroitDepart

Context Paquebot
inv: Paquebot.allInstances()->forAll(e1,e2|e1<>e2 implies checkNoConflict(e1.ListCroisiere,e2.ListCroisiere))

Justification

1. Nous avons séparé Vol/Itinéraire/Croisiere puisqu'ils sont très similaire mais conserve des différences notables du au mode de transport différent.
2. Puisque les recherches se font par compagnie et aeroport/gare/port, nous avons décidé de stocker les voyage séparément dans chaque objet dans une liste de voyage.
3. Pour s'assurer que la confirmation est faite sur le bon siège, on conserve la réservation courante dans l'objet Siege, et le l'objet client dans la réservation.
4. Notre Classe Système est associée(donc couplée) avec toute nos autres classes, mais par souci de clarté, nous n'avons pas mis aucun lien avec les autres classes.
5. Pour empêcher la double réservation d'un siège, nous avons choisi de conserver un objet réservation dans le siège et de vérifier avant la réservation qu'il n'en contient pas déja un.
6. Pour rechercher, notre système offre deux menus principaux, un pour les administrateurs et un autre pour les clients. Bien qu'ils semblent similaires, ils offrent des types de recherches différents.
