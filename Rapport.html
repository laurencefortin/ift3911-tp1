Hypothèses

  • Général
    • Pour les consultations, les clients peuvent seulement chercher par aéroport/port/gare
    • Pour les consultations des administeurs, il est possible de chercher par aéroport/port/gare mais également par compagnie.
    • Quand on arrive au menu, on choisi le type de voyage qu'on cherche pour ensuite choisir l'aéroport/port/gare pour finalement selectioner les autres critères
    • En se qui concerne le menu, il retourne une liste du choix prédéfini.
    • Même si les voyages férroviaires et aérien sont hautement similaire en terme de verification, on sépare quand même le menu en 2 sections comme le voyageur connait ses préférences.
    • Réservation de siège avec préférences est une classe
    • Pour des raisons éthiques, le systeme ne mémorise jamais les numéros de carte de crédit pour usage future.
    • Quand l'administeur recherche par compagnie, il doit choisir un choix du menu : Création, Modification, Supression
  • Voyage aérien:
    • Les sections des avions vont toujours être celles-ci :Première (F), Affaire (A), Économique Premium (P), Économique (E)
    • Une section est une classe
    • On affiche le nombre de siège disponible par section, dans un monde idéal, on affiche le layout de l'avion pour que l'utilisateur choisi le siege désiré
    • Il y a pas de limite dans les avions, quand on crer un vol l'aeroport associé s'assure de fournir un avion pour ce dit vol.
  • Voyage naval:
    • On assume que 1 cabine = 1 personne donc 1 réservation
    • Une section est une classe
    • Le port d'origine est le même que le port d'arrivé, c'est-à-dire qu'il n'y a pas de destination dans la recherche
  • Voyage ferroviaire:
    • Une ligne de train est une compagnie
    • Un trajet est un itinéraire
