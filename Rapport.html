Hypothèses

  • Idées Générales : Nous avons modifié quelque peu certaines étapes de l'utilisation du logiciel pour rendre l'expérience plus simple à l'Agent en réduisant la possibilité qu'il puisse faire un erreur. Par exemple, au lieu de devoir confirmer le Code de Service auquel l'Utilisateur veut s'inscrire, le logiciel lui affiche directement ceux qui sont disponibles. La même chose se produit lors d'une confirmation de participation, seulement les Services auxquels l'Utilisateur est inscrit sont affichés, etc.

  • Inscription : Les informations nécessaires à une inscription sont :
    • Numéro d'Utilisateur (généré automatiquement)
    • Nom
    • Prénom
    • Date de naissance
    • Adresse
    • Courriel
    • Numéro de Téléphone
    • Profesionnel ou non

  • Utilisateurs : Nous avons supposé qu'autant le Membre que le Professionnel pouvaientt être des Utilisateurs du #GYM, seulement le Professionnel accès à plus d'opérations dans le logiciel. Comme la création/consultation/modification des Services qu'il offre.

  • Numéro d'Utilisateur : Puisque ce n'était pas spécifié, nous avons supposé qu'un Membre et qu'un Professionnel utilisait le même type de numéro identifiant unique. Autant les Membres que les Professionnels doivent valider leur numéro d'Utilisateur avant d'accéder au #GYM ou toutes opérations du logiciel via l'Agent.

  • Code de Service : Numéro unique à 7 chiffres différent de ceux des Utilisateurs.

  • Services : Les Membres du #GYM peuvent seulement s'inscrire un Service la journée même.

  • Description des Services : Il est difficile pour un Humain d'identifier un Service seulement à l'aide d'un Code de 7 chiffres. C'est pourquoi nous avons pris la décision d'introduire une description au Service pour l'identifier plus facilement, celui-ci étant obligatoire contrairement au commentaire.
